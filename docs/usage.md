# 使用说明

## 1. 启动前准备

1. 确认系统为 Windows 10 或 Windows 11。
2. 已安装 Python 3.11 及以上版本，或使用后续发布的便携版。
3. 若需要使用 `AI 优化`，请准备有效的 SiliconFlow API Key。

## 2. 启动应用

### 源码运行

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python src/main.py
```

### 启动后的表现

- 程序会常驻系统托盘
- 首次未配置 API Key 时，会自动弹出 `LLM 设置`
- 双击托盘图标可打开历史面板

## 3. 主面板操作

主面板提供三块核心区域：

- 上方：历史列表
- 中间：可拖动的分隔条，用于调整上下比例
- 下方：当前项预览与动作按钮

可用操作：

- 双击历史项：复制并直接粘贴
- `复制`：将当前项复制回剪贴板
- `复制并粘贴`：复制后自动发送 `Ctrl+V`
- `AI 优化`：对当前文本执行 Prompt 优化
- 下拉框：选择要叠加的 Prompt 槽位；默认使用“默认 Prompt”
- `另存图片`：将图片历史导出为文件
- `清空历史`：清空当前会话内的文本与图片历史

## 4. Prompt 设置

托盘右键菜单中的 `Prompt 设置...` 用于控制 AI 优化策略。

包含三部分：

- 默认 Prompt：
  - 决定 AI 优化的基础规则
- 输出长度滑块：
  - 按原始输入的百分比表达大概长度约束
  - 默认值为 `300%`
  - 步进为 `50%`
  - 这是近似约束，不是严格的字符或 token 计算
- 自定义 Prompt 槽位：
  - 共 3 个槽位
  - 默认折叠
  - 只有“名称”和“Prompt 内容”都填写后，才会出现在主面板的下拉框中

如果用户在主面板里选中了某个槽位，程序会自动记住该选择，并在下次启动时继续使用。

## 5. LLM 设置

托盘右键菜单中的 `LLM 设置...` 用于配置模型调用参数。

可配置项包括：

- `API Key`
- `API Base`
- 模型名称
- `Temperature`
- 请求超时
- 是否启用 `max_tokens` 高级限制
- 是否启用思维链 / `CoT`

说明：

- 面向普通用户的长度控制在 `Prompt 设置` 中完成
- `max_tokens` 仅作为高级参数保留，默认关闭

## 6. AI 优化的工作方式

当前 `AI 优化` 采用“规则驱动，不做模板系统”的方式：

1. 内部通用规则负责约束输出质量
2. 默认 Prompt 作为主规则
3. 如果当前选择了槽位，则把槽位 Prompt 追加为附加规则
4. 最后自动加入输出长度百分比约束

目标是让模型根据输入内容自行发挥，而不是机械套模板。

## 7. 日志与排障

如果 `AI 优化` 失败，请优先查看 `ai_bridge.log`。

常见位置：

- 便携模式：程序目录下 `data/logs/ai_bridge.log`
- 普通模式：Windows 应用数据目录下 `WinClipboardAIEnhancer\\logs\\ai_bridge.log`

日志里会记录：

- 当前模型
- 超时配置
- 是否启用思维链
- 是否启用 `max_tokens`
- 输入长度
- 请求耗时
- HTTP 错误或网络异常

## 8. 当前已知限制

- 当前历史仍是会话内内存态，不做长期数据库存档
- `复制并粘贴` 依赖目标窗口当前仍可接收快捷键
- 图片操作目前以复制、预览、另存为主
- AI 优化效果与所选模型、网络质量和输入内容强相关
